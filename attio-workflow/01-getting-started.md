# Getting Started with Workflows (Attio Reference)

Source: https://attio.com/help/reference/automations/workflows/getting-started-with-workflows

## What are Workflows?

Workflows are visual automation tools that enable users to design powerful, intricate, and time-saving systems that can be iterated on demand. They support use cases ranging from task automation to lead scoring and data integration.

## Key Components

- **Canvas**: Primary workspace for piecing together workflow blocks
- **Blocks**: Individual actions or steps (sending emails, updating attributes, etc.)
- **Paths**: Connectors between blocks that dictate workflow sequence
- **Triggers**: Define when workflows start and what data enters the system
- **Inputs**: Data required for blocks to execute
- **Variables**: Dynamic values that change per workflow run (accessible via `{x}` icon)
- **Runs**: Individual workflow executions for tracking purposes
- **Credits**: Resource consumption based on workflow complexity
- **Templates**: Pre-built flows for quick starts
- **Notes**: Documentation features for workflows

## Building a Workflow (10-step process)

1. Choose starting point (blank or template)
2. Navigate the canvas
3. Select a trigger
4. Add blocks via the `+` icon
5. Configure inputs in the right panel
6. Use `{x}` icon for dynamic variables
7. Delete unwanted blocks with backspace
8. Reorder blocks by dragging connectors
9. Click "Publish" to activate
10. Toggle to pause workflows

## Running Workflows

Two execution methods:
- **Automatic**: Condition-based (record created, updated, scheduled, webhook)
- **Manual**: On-demand via Record command or Manual run triggers

## Access & Plans

- Available on all Attio plans
- Accessible to all workspace members
- Credit consumption varies by workflow complexity
- List access permissions required in settings for workflow-list integration
